Skill in action

Full-Stack Development

A working application slice using Node.js, Vue.js, Express, and MongoDB to support real CRUD workflows.

Scenario

A team needs a shared tool for creating, updating, searching, and validating records instead of maintaining spreadsheets.

Outcome

Converts manual record handling into a maintainable web workflow.

Real-World Business Example

A nonprofit tracks clients, services, appointments, and follow-ups in disconnected spreadsheets. I would build a web app where staff can search client records, create new service entries, update case notes, and filter follow-ups by due date. The Vue frontend would keep the workflow fast for staff, while Express routes would validate submissions and MongoDB would store flexible case records.

  • Interview users to define the record fields, statuses, and search paths they need daily.
  • Build API endpoints for creating, reading, updating, and filtering client activity.
  • Add form validation and clear error states so staff can trust the data they enter.

How I Apply It

  1. Design API endpoints around the actual user workflow.
  2. Build Vue components for fast record entry and review.
  3. Persist data with clear validation and predictable error handling.

Vue record editor
MongoDB collection model
